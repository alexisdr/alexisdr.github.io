05 · Demostración: impacto de un gran evento en el tráfico de Barcelona¶

Notebook de demostración end-to-end que carga el modelo LightGBM entrenado en el notebook 04 y simula el impacto en el tráfico de un partido de FC Barcelona en el Camp Nou:

  1. Carga el modelo (205 árboles, 96 features) y el dataset (629 713 registros horarios, 56 corredores).
  2. Define el evento simulado: FCB-Real Madrid, Champions League, 85 000 asistentes, kick-off 21:00 h.
  3. Selecciona como referencia el martes 2022-07-12 (51 corredores cubiertos, sin eventos).
  4. Construye el escenario "con evento" con peso de proximidad exponencial al Camp Nou (σ = 3.5 km).
  5. Predice sm (s/m) en ambos escenarios y calcula el delta de velocidad.
  6. Visualiza el impacto en un mapa folium interactivo y una tabla resumen por corredor.

    dia_semana= 1,         # martes (0=lunes)

)

# Distancia haversine (km) de cada corredor al Camp Nou
def haversine_km(lat1, lon1, lat2, lon2):
    R = 6371.0
    dlat = np.radians(lat2 - lat1)
    dlon = np.radians(lon2 - lon1)
    a = np.sin(dlat/2)**2 + np.cos(np.radians(lat1))*np.cos(np.radians(lat2))*np.sin(dlon/2)**2
    return R * 2 * np.arcsin(np.sqrt(a))

tram_geo['dist_campnou_km'] = haversine_km(
    tram_geo['lat_c'].values, tram_geo['lon_c'].values,
    EVENTO['lat'], EVENTO['lon']
)

# Peso de proximidad: decaimiento exponencial, mínimo 0.05 (toda la ciudad recibe algo)
tram_geo['proximity_w'] = np.maximum(0.05, np.exp(-tram_geo['dist_campnou_km'] / 3.5))

Se clonan las filas de referencia y se modifican las features de evento. Las features geo reales (dist_evento_min_km, inv_dist_evento, asistentes_pond, log_asistentes_pond, hay_evento_3km) se calculan con distancias haversine reales al Camp Nou para cada corredor — sin ponderación artificial:

hay_evento 0 1
n_eventos_agenda 0 1
log_n_eventos 0.0 0.693
asistentes_festival 0 85 000 × proximity_w
log_asistentes 0.0 log(1 + 85 000 × w)
evento_x_hora_sin/cos 0.0 sin/cos(2π·h/24)

log_asistentes_pond 0 log(1 + asistentes_pond)
hay_evento_3km 0 1 si dist ≤ 3 km

def prepare_X(frame):
    X = frame[FEATURES].copy()
    X['franja_horaria'] = X['franja_horaria'].astype('category')
    X['idTram']         = X['idTram'].astype('category')
    X['idItinerari']    = X['idItinerari'].fillna(-1).astype(int)
    return X

X_base = prepare_X(df_base)
X_ev   = prepare_X(df_ev)

pred_base = model.predict(X_base)
pred_ev   = model.predict(X_ev)

# Construir DataFrame de comparación

df_cmp['sm_base']    = pred_base
df_cmp['sm_evento']  = pred_ev
df_cmp['delta_sm']   = pred_ev - pred_base          # positivo = más lento
df_cmp['v_base']     = 3.6 / pred_base    # sm en s/m → v(km/h) = 3.6/sm

df_cmp['delta_kmh']  = df_cmp['v_evento'] - df_cmp['v_base']  # negativo = más lento

print(f'Corredores evaluados: {df_cmp["idTram"].nunique()}')
print(f'Delta sm medio:  {df_cmp["delta_sm"].mean():+.5f} s/m')

8. Mapa interactivo de impacto en los corredores¶

Los 51 corredores se colorean según el incremento medio de tiempo de viaje (delta_sm en s/m) a lo largo de la ventana 17–23 h:

  • Verde → sin impacto (Δsm ≈ 0)
  • Naranja / Rojo → mayor congestión (Δsm alto = tráfico más lento)

El grosor de la línea es proporcional al impacto. El marcador azul es el Camp Nou. El círculo de 3.5 km delimita la zona de mayor peso de proximidad.

El slider inferior permite avanzar de 17:00 a 23:00 en pasos de 1 hora. Los corredores se actualizan en cada paso mostrando el color e impacto correspondiente a esa hora concreta (verde → sin impacto, rojo → mayor congestión).

max_delta = float(df_cmp['delta_sm'].clip(lower=0).quantile(0.95))

# Función de color usando matplotlib (devuelve hex #rrggbb, compatible con Leaflet)
_cmap_mpl = mcm.RdYlGn_r                                  # verde → amarillo → rojo
_norm      = mcolors.Normalize(vmin=0, vmax=max_delta)

def colormap(delta):
    """Devuelve color hex (#rrggbb) para un valor delta_sm dado."""
    rgba = _cmap_mpl(_norm(max(0.0, min(float(delta), max_delta))))
    return mcolors.to_hex(rgba)

import datetime
from folium.plugins import TimestampedGeoJson

BASE_DT = datetime.datetime(2022, 10, 4)  # día del partido simulado

# ── Construir features GeoJSON: una por (corredor × hora) ────────────────────
features_tiempo = []
for hora in EVENTO['horas']:
    ts = (BASE_DT + datetime.timedelta(hours=hora)).isoformat()

    df_hora = (
        df_cmp[df_cmp['hora_dia'] == hora]
        .merge(
            tram_geo[['idTram', 'lat_ini', 'lon_ini', 'lat_fi', 'lon_fi',
                       'desc', 'dist_campnou_km']],
            on='idTram', how='left'
        )
    )

    for _, row in df_hora.iterrows():
        if pd.isna(row['lat_ini']):
            continue
        delta = max(0.0, row['delta_sm'])
        color  = colormap(delta)
        weight = 4 + 4 * (delta / max(max_delta, 1e-6))

        features_tiempo.append({
            'type': 'Feature',
            'geometry': {
                'type': 'LineString',
                'coordinates': [
                    [row['lon_ini'], row['lat_ini']],
                    [row['lon_fi'],  row['lat_fi']],
                ]
            },
            'properties': {
                'times': [ts, ts],   # mismo instante en ambos extremos → aparece completo
                'style': {
                    'color':   color,
                    'weight':  round(weight, 1),
                    'opacity': 0.88,

Interpretación del modelo:

  • El modelo detecta el evento y redistribuye el tráfico correctamente: el pico de congestión se produce a las 20:00, 1 hora antes del kick-off, capturando la afluencia máxima.
  • Los corredores más afectados (Muntaner × 2, Riera Blanca, Pg. Manuel Girona, Balmes × 3) son vías de acceso típicas al Camp Nou desde el norte y noreste de la ciudad.
  • La señal de asistentes_festival × proximity_w genera diferenciación espacial: corredores a <1 km reciben >77 % de la señal de 85 000 asistentes.
  • A partir de las 22:00 el impacto se disipa, coherente con el éxodo post-partido que en el período de referencia era bajo al tratarse de una noche de verano sin evento.
