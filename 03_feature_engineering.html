03 · Feature Engineering¶

A partir del dataset integrado (dataset_integrado_horario.parquet) se construyen las variables de entrada para el modelo de predicción espacio-temporal. La variable objetivo es sm (segundos por metro), la velocidad invertida normalizada por la longitud del tramo, que permite comparar directamente corredores de distintas dimensiones.

Bloques de features¶

Bloque Variables generadas
1 · Lag features Valores pasados de sm por tram (1h–168h) + lags de tiempo de viaje
2 · Rolling windows Medias, desv. estándar y máximos deslizantes de sm por tram
3 · Features derivadas Delta tiempo, tendencia acumulada, rush-hour flags
4 · Interacciones evento × hora Productos cruzados entre hay_evento (flag unificado FCB + conciertos + ferias + festivales) y contexto temporal
5 · Validación Diagnóstico de nulos, distribuciones, correlaciones con sm

1 · Lag features¶

Para cada tram, se desplazan hacia adelante los valores históricos de sm (velocidad invertida normalizada) y el tiempo de viaje actual. Los lags capturan la autocorrelación temporal: el sm de hace 1 hora o hace 7 días es un predictor fuerte del sm actual.

Lag Razonamiento
1 h, 2 h, 3 h Inercia a corto plazo
6 h, 12 h Patrón de media jornada
24 h Mismo momento del día anterior
48 h Anteayer
168 h Misma hora de la semana pasada

Nota: se agrupan por idTram para no mezclar lags entre trams distintos.

2 · Rolling windows¶

Las ventanas deslizantes suavizan el ruido puntual y exponen tendencias de sm a distintas escalas temporales. Se usa min_periods=1 para no perder filas cuando el tram tiene menos datos que el tamaño de la ventana.

3 h media
6 h media, desv. estándar
12 h media
24 h media, desv. estándar, máximo
168 h media (patrón semanal)

def rolling_by_tram(series: pd.Series, window: int, func: str, min_p: int = 1) -> pd.Series:
    'Rolling statistic agrupado por idTram, basado en un offset de posiciones.'
    return (
        series
        .groupby(df['idTram'], sort=False)
        .transform(lambda x: getattr(x.shift(1).rolling(window, min_periods=min_p), func)())
    )

# Media deslizante de sm
for w in [3, 6, 12, 24, 168]:
    df[f'sm_rmean_{w}h'] = rolling_by_tram(df['sm'], w, 'mean')

# Desviación estándar (volatilidad de sm)
for w in [6, 24]:
    df[f'sm_rstd_{w}h'] = rolling_by_tram(df['sm'], w, 'std', min_p=2)

# Máximo deslizante de sm (pico reciente de congestión)
df['sm_rmax_24h'] = rolling_by_tram(df['sm'], 24, 'max')

# Diferencia media-corto vs media-largo (momentum de sm)
df['sm_momentum'] = df['sm_rmean_3h'] - df['sm_rmean_24h']

roll_cols = [c for c in df.columns if 'rmean' in c or 'rstd' in c or 'rmax' in c or 'momentum' in c]

4 · Interacciones evento × hora¶

El impacto de un evento sobre el tráfico no es uniforme: un evento afecta más a las 20 h que a las 7 h, y más en fin de semana que en miércoles. Estas interacciones permiten al modelo aprender estos patrones diferenciados.

Se generan productos cruzados entre:

  • hay_evento — flag unificado FCB + conciertos (Palau Sant Jordi) + ferias (Fira) + festivales/fiestas
  • Contexto temporal (hora cíclica, día de semana, rush-hour)

# ── Haversine vectorizada ─────────────────────────────────────────────────────
def haversine_km(lat1, lon1, lat2, lon2):
    R = 6371.0
    dlat = np.radians(np.asarray(lat2, float) - lat1)
    dlon = np.radians(np.asarray(lon2, float) - lon1)
    a = (np.sin(dlat/2)**2
         + np.cos(np.radians(lat1)) * np.cos(np.radians(np.asarray(lat2, float)))
         * np.sin(dlon/2)**2)
    return R * 2 * np.arcsin(np.sqrt(np.clip(a, 0, 1)))

7 · Conclusiones — Idoneidad del dataset para el modelado¶

7.1 Fortalezas del dataset¶

Autocorrelación temporal extraordinariamente alta El ranking de correlaciones y el sanity check confirman que sm tiene autocorrelación r = 0,978 a 1 hora y r = 0,966 a 168 horas (7 días). El sanity check en tram 78 lo cuantifica directamente: MAE naive lag_1h = 0,062 s/m ≈ MAE naive lag_168h = 0,063 s/m — la semana anterior predice tan bien como la hora anterior. Este nivel es muy superior al obtenido con factor_medio (r ≈ 0,59) porque sm no tiene el efecto de suelo del 78,9 % de registros en factor = 1,0.

Variable objetivo sm normalizada y con amplio rango sm = tiempo_actual / longitud_tramo_m (s/m): mediana 0,3023 s/m ≈ 11,9 km/h, rango 0,009–2,828 s/m. Permite comparar directamente corredores de 247 m a 1.231 m sin sesgo de longitud. Interpretable: 0,28 s/m ≈ 13 km/h (fluido), 0,50 s/m ≈ 7,2 km/h (congestionado).

Cobertura temporal H2 2021 + 2022 (H2 2021 como contexto rodante) La ventana 2021-07-01 → 2022-12-31 con 12.778 horas únicas por tram proporciona suficientes ejemplos para capturar estacionalidad anual y patrones semanales estables. 2022 es el primer año completo post-pandemia en Barcelona — el dataset más limpio metodológicamente.

Tipo de vía y longitud de tramo bien representadas Los 58 trams cubren Diagonal (8), Avinguda (13), Passeig (2), Via (6), Travessera (1) y Carrer (28). La diversidad morfológica permite al modelo aprender diferencias estructurales de capacidad.

Flag hay_evento unificado con interacciones 181 fechas únicas con evento (FCB + conciertos + ferias + festivales). Las 22 features de interacción (hay_evento × temporal) permiten a modelos no lineales capturar el impacto diferencial por hora, día y corredor.

7.3 Conclusión¶

El dataset de features cumple los requisitos para entrenar y evaluar modelos de predicción de sm:

  • Volumen: 629.713 filas × 107 columnas — suficiente para Random Forest, XGBoost y LightGBM
  • Señal predictiva dominante: autocorrelación muy alta (r > 0,96 para todos los lags principales) — el modelo de árbol superará fácilmente el baseline naive
  • Variable objetivo sm: sin nulos, rango amplio, interpretable en km/h, sin sesgo de suelo
