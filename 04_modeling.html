04 · Modelado — Predicción de sm (tiempo de recorrido normalizado)¶

A partir del dataset de features (dataset_features.parquet) se entrena y evalúa un modelo de predicción de sm = tiempo_actual / longitud_tramo_m (s/m) por (idTram, hora).

El baseline trivial (mediana → MAE=0.1862) confirma que para sm no existe la trampa del 95% concentrado en 1.0 de factor_medio: predecir siempre la mediana tiene error muy elevado.

Hallazgo clave: el lag 1h (MAE=0.0171 s/m) es mejor que el lag 168h (MAE=0.0200 s/m) por un 17%. Para factor_medio estos valores eran casi idénticos (ratio ~1.0×), pero con sm la variación intra-diaria es mayor que la reproducibilidad semanal exacta, dando ventaja al lag de corto plazo. Cualquier modelo entrenado debe conseguir MAE < 0.0171 s/m para justificar su complejidad.

3 · Entrenamiento LightGBM¶

Se usa LightGBM con los siguientes hiperparámetros de partida, elegidos para un primer modelo razonablemente bueno sin tuning exhaustivo:

  • num_leaves=127 — árbol moderadamente complejo sin sobreajuste
  • learning_rate=0.05 — conservador para que el early stopping encuentre el óptimo
  • min_child_samples=50 — regularización mínima por hoja
  • early_stopping_rounds=50 — para en validación si no mejora en 50 rondas

# Construir datasets LightGBM con soporte para categoricals
cat_cols_idx = [FEATURES.index(c) for c in CAT_FEATURES if c in FEATURES]
cat_itinerari_idx = [FEATURES.index('idItinerari')] if 'idItinerari' in FEATURES else []
all_cat_idx = cat_cols_idx + cat_itinerari_idx

dtrain = lgb.Dataset(X_train, label=y_train,
                     categorical_feature=all_cat_idx, free_raw_data=False)
dval   = lgb.Dataset(X_val,   label=y_val,
                     categorical_feature=all_cat_idx, free_raw_data=False,
                     reference=dtrain)

params = {
    'objective':         'regression',
    'metric':            'mae',
    'num_leaves':        127,
    'learning_rate':     0.05,
    'feature_fraction':  0.8,
    'bagging_fraction':  0.8,
    'bagging_freq':      5,
    'min_child_samples': 50,
    'lambda_l1':         0.1,
    'lambda_l2':         0.1,
    'verbose':           -1,
    'n_jobs':            -1,
    'seed':              42,
}

callbacks = [
    lgb.early_stopping(stopping_rounds=50, verbose=True),
    lgb.log_evaluation(period=100),
]

print('Entrenando LightGBM...')

    params,
    dtrain,
    num_boost_round=2000,
    valid_sets=[dval],

def evaluar(y_true, y_pred, nombre):
    mae  = mean_absolute_error(y_true, y_pred)
    rmse = mean_squared_error(y_true, y_pred) ** 0.5
    r2   = r2_score(y_true, y_pred)
    return {'split': nombre, 'MAE': mae, 'RMSE': rmse, 'R2': r2, 'n': len(y_true)}

resultados = []
for X, y, nombre in [(X_train, y_train, 'Train'),
                     (X_val,   y_val,   'Val'),
                     (X_test,  y_test,  'Test')]:
    pred = model.predict(X, num_iteration=model.best_iteration)
    resultados.append(evaluar(y, pred, nombre))

res_df = pd.DataFrame(resultados).set_index('split')

5 · Importancia de features — SHAP¶

SHAP (SHapley Additive exPlanations) descompone cada predicción en la contribución marginal de cada feature. A diferencia del feature importance por ganancia de LightGBM, SHAP es consistente y comparable entre features.

    n_excluidos = (~mask_clean).sum()
    mae_clean   = df_clean['error_abs'].mean()
    rmse_clean  = np.sqrt(((df_clean[TARGET] - df_clean['pred']) ** 2).mean())
    r2_clean    = r2_score(df_clean[TARGET], df_clean['pred'])

    print(f'Corredors con MAE > 3× media excluidos: {outliers_excluir}')
    print(f'MAE TEST global (todos):                {mae_test_model:.4f}  (n={len(df_test):,})')
    print(f'MAE TEST limpio (sin {outliers_excluir}): {mae_clean:.4f}  (n={len(df_clean):,})')
    print(f'RMSE TEST limpio: {rmse_clean:.4f}')
    print(f'R²   TEST limpio: {r2_clean:.4f}')
    print(f'Registros excluidos: {n_excluidos:,} ({100*n_excluidos/len(df_test):.1f}%)')
else:
    print('No hay corredors con MAE > 3× la media global. Sin exclusiones.')
    mae_clean = mae_test_model

7.2 Decisiones metodológicas adoptadas¶

1. TARGET = sm (no factor_medio) sm = tiempo_actual / longitud_tramo_m (s/m) elimina el efecto techo del 95.3% en 1.0. Autocorrelación r=0.978 (lag 1h) vs r≈0.59 de factor_medio. Los baselines son mucho más exigentes en unidades reales de velocidad.

2. Exclusión de leakage intra-hora (16 columnas) tiempo_actual y todas sus derivadas (factor_medio, delta_tiempo, ratio_tiempo, tendencia_media, sm_diff_*, tipo_via) excluidas. Las 96 features restantes (91 previas + 5 geo: dist_evento_min_km, inv_dist_evento, asistentes_pond, log_asistentes_pond, hay_evento_3km) están todas calculadas sobre instantes anteriores al slot predicho.

3. Split semestral H2 2021 / H1 2022 / H2 2022 Splits equilibrados (~202–216 K filas). El train y test cubren el mismo semestre (H2), lo que reduce la deriva estacional: MAE test (0.0078) < MAE val (0.0098).

4. Baselines sobre el período de test exclusivamente Asegura comparación justa: el modelo y los baselines se evalúan sobre los mismos datos.

5. Segmentación fluido/congestionado (P75 = 0.4618 s/m) Separación del tráfico normal del congestionado — escenario de interés del TFM.

7.3 Hallazgos clave¶

Rendimiento general: R²=0.991 y 54.4% de mejora sobre el lag 1h confirman que el modelo extrae información real de las features de eventos, meteorología y contexto espacial, más allá de la autocorrelación de primer orden.

Señal de eventos (resultado central del TFM): El modelo predice 14% mejor las horas con evento activo (MAE 0.0072 vs 0.0083). En las horas congestionadas, la mejora con evento es del 14% (0.0117 vs 0.0136): el modelo aprovecha hay_evento precisamente cuando la predicción tiene mayor valor operativo. Esto responde afirmativamente la hipótesis del TFM: los eventos urbanos permiten mejorar la predicción de tráfico de forma medible y estadísticamente significativa.

Franjas horarias: mediodia tiene el mayor error (MAE=0.0095), superando a tarde (0.0080). La variabilidad imprevisible del mediodía en H2 2022 (eventos gastronómicos, ferias, hora de comer) es más difícil de capturar que los ciclos regulares de mañana/tarde.

Corredores: Los corredores 11 y 14 (MAE~0.022, 2.7–2.8× media) son tramos rápidos con alta variabilidad relativa, no distributional shift. Ningún corredor supera 3× MAE global: el modelo generaliza uniformemente sobre todos los tramos de la red monitorizada.
